# 📦 Deliverex Bot

Телеграм-бот для чата логистической компании Deliverex (доставка грузов из Китая в РФ).

**Что умеет:**

1. **Приветствие новичков** — при входе в группу отправляет согласованный текст с просьбой прислать категорию товара, вес и объём.
2. **Приём заявок** — распознаёт параметры груза в сообщении, сохраняет заявку в БД и мгновенно присылает карточку менеджеру (@kurotoplol) — с именем клиента-ссылкой, по которой открывается диалог с ним.
3. **Антиспам** — удаляет рекламу конкурентов, ссылки, приглашения «пишите в ЛС», флуд и повторы. Работает по балльной системе: предупреждение → мут → бан. Словарь стоп-слов пополняется прямо из чата, без редеплоя.
4. **Модерация вручную** — команды `/warn`, `/mute`, `/ban`, `/trust` и др. для владельца и админов.

## 1. Быстрый старт (Railway)

### Шаг 1. Создать бота

1. Написать [@BotFather](https://t.me/BotFather) → `/newbot` → получить токен.
2. Там же: `/setprivacy` → выбрать бота → **Disable**.
   ⚠️ Это обязательно: иначе бот не увидит сообщения в группе и не сможет ни ловить спам, ни принимать заявки.

### Шаг 2. Развернуть на Railway

1. Залить эту папку в репозиторий GitHub.
2. Railway → **New Project** → **Deploy from GitHub repo** → выбрать репозиторий.
3. В том же проекте: **New** → **Database** → **Add PostgreSQL**.
4. Открыть сервис с ботом → вкладка **Variables** → **Raw Editor** → вставить переменные из `.env.example`, подставив свои значения. Для базы указать именно так:

   ```
   DATABASE_URL=${{Postgres.DATABASE_URL}}
   ```

5. Deploy. Таблицы создадутся автоматически при первом запуске (`schema.sql` применяется идемпотентно).

> Бот работает в режиме **long polling** — публичный домен и healthcheck не нужны. Если Railway предлагает сгенерировать домен, это можно пропустить.

### Шаг 3. Настроить группу

1. Создать **супергруппу**, включить **Темы (Topics)**.
2. Создать темы: `Болталка`, `Закупка / подбор товара`, `Оплата в Китай`, `Обсуждение логистики`, `Отзывы`.
3. Добавить бота в группу и **выдать права администратора**:
   - Удаление сообщений ✅
   - Блокировка пользователей ✅
   - (остальное не требуется)
4. Отправить в группе `/id` → скопировать `Chat ID` (вида `-1001234567890`) в переменную `GROUP_CHAT_ID` на Railway.
5. Скопировать текст из `bot/texts.py` → `GROUP_DESCRIPTION` в описание группы.

### Шаг 4. Подключить менеджера

Менеджер (@kurotoplol) один раз пишет боту в личку `/start` — бот запомнит его `chat_id` и будет слать туда все заявки.

Альтернатива: узнать его числовой ID и прописать в `MANAGER_CHAT_ID`.

Заявки можно направлять и в отдельный служебный чат — добавить туда бота и отправить `/setmanager`.

### Шаг 5. Скрыть участников

Требование «в чате не должно быть видно участников» — это **настройка Telegram, не бота**:
Управление группой → **Участники** → **Кто может видеть участников** → **Только администраторы**.

## 2. Локальный запуск

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
copy .env.example .env         # заполнить BOT_TOKEN и DATABASE_URL
python -m bot.main
```

## 3. Команды

**Для клиентов**

| Команда | Действие |
|---|---|
| `/start` | приветствие и инструкция (в личке) |
| `/zayavka <текст>` | оставить заявку вручную |
| `/help` | как это работает |

**Для админов и менеджеров**

| Команда | Действие |
|---|---|
| `/leads [N]` | последние N заявок |
| `/stats` | статистика по чату и заявкам |
| `/setmanager` | слать заявки в текущий чат |
| `/warn [причина]` | предупреждение (ответом на сообщение) |
| `/unwarn` | снять предупреждения |
| `/mute [минуты]` | ограничить отправку сообщений |
| `/unmute` | снять ограничение |
| `/ban` · `/unban` | блокировка |
| `/trust` · `/untrust` | вывести пользователя из-под антиспама |
| `/spam_list` | показать словарь антиспама |
| `/spam_add <фраза>` | добавить стоп-слово |
| `/spam_add_re <regex>` | добавить regex-правило |
| `/spam_del <ID>` | отключить правило |
| `/id` | ID чата, темы и пользователя |
| `/welcome` | показать текущее приветствие |

## 4. Как работает антиспам

Каждое сообщение от неадмина получает баллы:

| Признак | Баллы |
|---|---|
| Ссылка на t.me / tg:// | 3 |
| Внешняя ссылка | 2 (3 у новичка) |
| Скрытая ссылка в разметке | 3 |
| Пересланное сообщение | 2 (3 у новичка) |
| Номер телефона | 2 |
| Стоп-слово из словаря | 2–4 |
| «Пишите в ЛС» (по смыслу) | 3 |
| Флуд (5 сообщений за 10 сек) | 3 |
| Повтор одного сообщения | 3 |
| Скрытые символы (обход фильтра) | 2 |
| Избыточный капс | 1 |

Набрал `SPAM_SCORE_THRESHOLD` (по умолчанию 3) → сообщение удаляется, выдаётся предупреждение.
`WARN_LIMIT` предупреждений (3) → мут на `MUTE_MINUTES` (60 мин).
`BAN_AFTER_MUTES` мутов (3) → бан.

Движок устойчив к обходам: латиница вместо кириллицы (`пиwите в лc`) и невидимые символы нормализуются перед проверкой.

Все настройки меняются переменными окружения без правки кода. Словарь пополняется командой `/spam_add` — изменения подхватываются в течение минуты.

## 5. Структура проекта

```
bot/
  main.py            запуск, роутеры, middleware
  config.py          чтение переменных окружения
  db.py              работа с PostgreSQL (asyncpg)
  texts.py           ВСЕ тексты бота — правки здесь
  antispam.py        балльный движок антиспама
  leads.py           разбор заявок (категория, вес, объём)
  middlewares.py     учёт пользователей + фильтр спама
  utils.py           права, безопасные вызовы API, доставка заявок
  handlers/
    common.py        /start, /help, /id, /setmanager
    welcome.py       вход и выход участников
    leads.py         приём заявок
    moderation.py    модерация и словарь антиспама
schema.sql           схема БД
.env.example         шаблон переменных окружения
```

Тексты приветствия и ответов клиенту вынесены в `bot/texts.py` — заказчик сможет их корректировать без изменения логики.
